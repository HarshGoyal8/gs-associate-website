# GS Associate Website — Sprint 05

> **Sprint:** 05 — Backend Integration
> **Version:** 0.5.0
> **Status:** Planned
> **Estimated Duration:** 7–10 Days

---

# Sprint Goal

Transform the website from a static frontend into a dynamic application by integrating Supabase and Cloudinary.

All product, brand, catalogue, and enquiry data should now come from the backend instead of hardcoded files.

---

# Sprint Objectives

* Integrate Supabase
* Configure Cloudinary
* Build database schema
* Connect frontend to backend
* Store media externally
* Prepare for CMS integration

---

# Features

## Supabase Setup

* Create Supabase project
* Configure environment variables
* Connect application
* Test database connection

---

## Database

Create the following tables:

* Products
* Categories
* Brands
* Catalogues
* Testimonials
* Dealer Enquiries
* Website Settings

---

## Cloudinary

Configure:

* Image Uploads
* PDF Uploads
* Image Optimization
* Responsive Images

## Dynamic Content

Replace hardcoded data with database-driven content for:

* Homepage
* Products
* Brands
* Catalogues
* Testimonials

---

## Forms

Connect:

* Contact Form
* Dealer Registration Form

Store submissions inside Supabase.

## API Layer

Develop reusable services for:

* Products
* Brands
* Categories
* Catalogues
* Testimonials
* Enquiries

## Error Handling

Implement:
